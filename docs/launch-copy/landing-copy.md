## FAQ（6 条）

**Q1：什么是 BYO API key？为什么要这么做？**

> Bring-Your-Own API key — 你自己去 X 开发者后台、Reddit OAuth 后台拿到 key/token，填给 Postwave，我们帮你调。
>
> 为什么不替你代办？两个原因：
> 1. 平台越来越严，第三方代发是封号高发区。你自己的账号、自己的 token，最安全。
> 2. X 的 API 已经收费了（$0.01/post 起步），我们如果替你买，要么涨订阅费要么限制频率，都不如你自己控成本。
>
> 整个 OAuth 流程我们做了 step-by-step 引导，不懂技术也能 5 分钟配完。

**Q2：我的账号会被封吗？**

> 海外平台用官方 OAuth + 官方 API，按官方速率限制走，被封概率极低（和你自己手动发一样的渠道）。
>
> 中国平台（即刻 / V2EX）走 Electron 容器代理，模拟真实浏览器行为；但任何自动化都有非零封号风险，我们的对策：
> - 频率限制：默认每平台每天最多 3 帖
> - 单账号隔离：你的 cookie 不和别人共享
> - 你可以随时切换为"只生成稿、我手动复制粘贴去发" 模式
>
> 我们不替你保证不被封，只承诺：风险可控、出问题立即停。

**Q3：现在能用吗？**

> 还不能。
>
> Postwave 在 Phase 0 — landing + waitlist 阶段，预计 2026 年 Q3 内测开放。这个 landing 本身也是用 Postwave 的方式（手动 + AI 辅助）推到 7 个平台的，你正在看的就是 dogfood 第 0 个版本。
>
> 你加入 waitlist 后，我会用邮件公开 build 进度（每 2 周一封，含数据），内测开放时早鸟先进。

**Q4：海外和中国发布有什么差异？**

> 海外（Reddit / X / Threads / Dev.to 等）走官方 OAuth，几乎所有平台 API 都很成熟，发布 1 秒到位、数据回收实时。
>
> 中国（即刻 / V2EX）没有官方写入 API，我们用 Electron + Playwright 跑在阿里云 ECS 上，模拟你浏览器行为发布；速度慢一点（每帖 5-10 秒），数据回收比海外慢半小时。
>
> Phase 1 v0 中国轨**只做即刻 + V2EX 两个**。小红书 / B 站 / 知乎反爬太强 + 政策敏感，我们只给你生成稿，让你手动去发。

**Q5：我的 API key 和数据安全吗？**

> 关于 key：所有 OAuth token 都在 Postgres 用 AES-256 加密存储，加密 key 在环境变量里、不入库。我会发一份每月的 SOC 2 lite 自检报告（Phase 1 上线后）。
>
> 关于内容：你的产品 URL、生成的文案、发布数据 — 都是你的。我们不会拿来训练模型、不会卖给第三方、不会用于其他用户的内容生成参考。
>
> 关于代码：Phase 1 上线后会开源 platforms adapter 部分（学 Postiz 模式），让你可以自托管，进一步降低信任成本。

**Q6：Roadmap 是什么？**

> - Phase 0（现在 → 2 周内）：landing + waitlist + dogfood 推自己
> - Phase 1（4-6 周）：MVP — 海外 10 + 中国 2 平台直发 + AI 内容生成 + 数据回收
> - Phase 2（3-6 个月后）：需求池接入 — 爬 Reddit / 即刻 / V2EX 自动发现独立开发者机会，从 launcher 升级为"OPC 操作系统"
>
> 我不打算做 affiliate / 分销跟踪 / revenue tracking — 那些用 Stripe Atlas 自己解决就够，不是核心痛点。
