# 品牌命名候选 — OPC LaunchOS

> 调研日期：2026-04-30
> 域名可用性核实方式：
> - `.so` 通过 `whois -h whois.nic.so` 直查 sonic 注册局（"No Object Found" = 可用）
> - `.com / .ai / .dev` 通过 `dig NS` + 交叉对比 whois（有 DNS 记录 = TAKEN）
> - `.com` 几乎全部 TAKEN — indie SaaS 命名现状已是这样，不必勉强。优先 `.so` / `.ai`

---

## 候选 1 — OPC.Dino（中文：OPC.Dino / 推浪）

**含义**
- "Post"（发帖）+ "Wave"（潮、波）= 一篇内容像一道浪，被推到所有平台。
- 也呼应"浪潮 / build in public 浪潮"。

**与产品定位的关系**
正中红心：一个产品 = 一道波，向多个平台同时扩散。比"scheduler"（排程）更有动作感，比"launcher"（启动器）更轻松。中文译名"OPC.Dino"自带社交气质，能在即刻 / V2EX 这类中文圈直接用。

**域名可用性（实测）**
- `opcdino.com` — TAKEN
- `opc.dinolabs.online` — **可用** ✅
- `opcdino.ai` — TAKEN
- `opcdino.dev` — TAKEN

**优势**
1. 名词 + 动作意象兼具，落到 logo 容易（一道波 / 一个发射符号）
2. 发音简单，海外中文圈都好读

**短板**
1. `.com` 已被占（但本来 `.com` 也基本不奢望）
2. "wave" 在 SaaS 命名里不算稀缺（有 Wavebox 等），有撞衫风险

---

## 候选 2 — OPClaunch（中文：一人发射台 / OPC 发射）

**含义**
- 用户 PLAN.md 原始代号"OPC 操作系统"中的 OPC（一人公司 / One-Person Company）+ launch。
- 直接告诉用户："这是为 OPC（一人公司）做的 launch 工具。"

**与产品定位的关系**
最直白的 ICP 命名。和"Indie Hackers"那种社区品牌一脉相承。但它锁死了"一人公司"这个细分，未来如果想扩到 2-5 人小团队会有点尴尬。

**域名可用性（实测）**
- `opclaunch.com` — TAKEN
- `opclaunch.so` — **可用** ✅
- `opclaunch.ai` — **可用** ✅
- `opclaunch.dev` — 未查（不重要）

**优势**
1. 和用户沉淀的 "OPC 操作系统" 概念延续，复用前面所有研究内容
2. 中文圈"一人公司"概念正在升温，自带话题

**短板**
1. "OPC" 在中文里是陌生缩写，要花 1-2 句话解释（即刻发文每次都要科普，长期成本高）
2. 国际市场上 OPC 是"OLE for Process Control"（工业协议），SEO 容易污染

---

## 候选 3 — ShipIndie（中文：发船 / 出海）

**含义**
- "Ship"（独立开发者圈最强的动词："Ship it"）+ "Indie"（独立）。
- 中文双关：发船 = 出海，正好对应"出海中国 indie"这个核心 ICP。

**与产品定位的关系**
直接绑死 ICP（独立开发者）+ 行为（发布）。"出海"在中国 indie 圈是高情感词。海外 "ship it" 文化也认。

**域名可用性（实测）**
- `shipindie.com` — TAKEN
- `shipindie.so` — **可用** ✅
- `shipindie.ai` — **可用** ✅

**优势**
1. ICP + 动作绑死，传播一句话讲清楚
2. 中英双关（ship → 出海），中国轨海外轨都不别扭

**短板**
1. 含 "indie" 字眼会把"小团队"用户挡在门外（虽然 ICP 就是 indie，但产品成长后会撞墙）
2. "Ship" 在英文 SaaS 命名里被滥用（Shipfast / Shipped / Shipright / Shipfox），辨识度一般

---

## 候选 4 — Lighthous（中文：灯塔 / 引航）

**含义**
- Lighthouse 砍掉尾 e，避开 Google Lighthouse 的 SEO 污染。
- 灯塔 = 你的产品 = 我帮你把光打到所有海岸（平台）。

**与产品定位的关系**
适合做"引导用户找到产品"的叙事 — 这个比"发"更高级，未来接 Phase 2（需求池）时也讲得通："你是灯塔，我帮你看见远方有谁在找你。" 是少有的能同时撑两个 phase 的名字。

**域名可用性（实测）**
- `lighthous.com` — TAKEN
- `lighthous.so` — **可用** ✅
- `lighthous.ai` — TAKEN
- 完整拼写 `lighthouse.*` 几乎都 TAKEN

**优势**
1. 意象最美，最容易做品牌（logo 就是一座灯塔 + 光束分到 N 个平台）
2. 故事可以撑到 Phase 2（需求池 = 看见远方信号）

**短板**
1. 拼写不完整（缺 e），用户口口相传容易拼错成 lighthouse.com 跳错地方
2. 与 Google Lighthouse（性能审计工具）撞名严重，长期 SEO 是噩梦

---

## 候选 5 — AllChannels（中文：全频道 / 全渠道）

**含义**
- All Channels = 所有渠道（平台）。最功能性、最直白。

**与产品定位的关系**
就像 Buffer 的"all-in-one social"那样直接。零想象力，但零误解。

**域名可用性（实测）**
- `allchannels.com` — TAKEN
- `allchannels.so` — **可用** ✅
- `allchannels.ai` — **可用** ✅

**优势**
1. 自解释，连 landing 文案都省了 — 用户看到名字就懂干啥
2. 关键词 SEO 友好（"channel" 是这类工具的高频搜索词）

**短板**
1. 太通用没记忆点 — 投资人 / 用户读完转头就忘
2. 和 Buffer / Hootsuite 显得是同一品类，反而冲淡了"OPC 操作系统"这个差异化

---

## 推荐

**首选：OPC.Dino+ `opc.dinolabs.online`**

**理由（三条）**：

1. **意象 × 动作 × 中英通吃** — "Post"（发）+ "Wave"（浪）一秒讲清产品做什么；中文译名"OPC.Dino"在即刻 / V2EX 自然，海外读法也舒服。其他候选要么锁死 ICP（OPClaunch / ShipIndie 撞墙），要么 SEO 受污染（Lighthous），要么过于通用（AllChannels）。

2. **域名拿得到一个干净的 `.so`** — `.com` 已普遍是 reseller 高价标的，`opc.dinolabs.online` 在 sonic 注册局 `No Object Found`，年费 $30-40 即可拿下。对 indie SaaS 这就是实际能买的最好结果。`.so` 还自带 "Software" 双关。

3. **不锁死 ICP，但又有方向** — 不像 OPClaunch / ShipIndie 把"一人公司""indie"焊死在品牌里，OPC.Dino 既能讲 indie 故事，将来扩到 5 人小团队 / agency 也不别扭。Phase 2 需求池接入时也能解释成"先看到浪从哪来，再决定推哪道浪"。

**推荐域名组合**：
- 主域名：`opc.dinolabs.online`（产品官网）
- 备选可一起注册：`opcdino.io`（如可用，做技术文档）/ `opcdino.app`（做应用入口）

**次选**（如果觉得 OPC.Dino 太"营销味"）：**OPClaunch + `opclaunch.ai`**
- 优势：和用户已有研究沉淀（"OPC 操作系统"）天然连续；`.ai` 还能拿到
- 代价：永远要给用户解释什么是 OPC

---

## 不推荐的原候选说明

- **Buoy / Voyage** — 域名层面已经基本拿不到（buoy.com / voyage.com 都是老域名，buoy.so 在 2026-04 刚被抢注，buoy.ai 也被占）。即使能拿到非主流后缀，"浮标 / 航行"这种隐喻太隐晦，需要文案大量解释，不适合 indie 速度。
- **BuildIO** — `buildio.com` 早被占；"io" 后缀疲劳；含义比 OPC.Dino 更弱（build 太泛）。
- **Indie Launch OS** — 三个词太长不好做 logo，SEO 也撒胡椒面。
